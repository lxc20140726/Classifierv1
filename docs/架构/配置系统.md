# 配置系统设计 v3.1

> 版本：v3.1 | 日期：2026-03-28

## 目标

- 同时兼容 Win / macOS / Docker(Linux) 的路径语义
- 保持 `app_config` 与 legacy `config` KV 一致
- 防止部分字段更新导致配置被误清空
- 明确扫描目录来源优先级

## 存储模型

```sql
CREATE TABLE app_config (
  id INTEGER PRIMARY KEY CHECK (id = 1),
  version INTEGER NOT NULL,
  value TEXT NOT NULL,
  checksum TEXT NOT NULL,
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

- `app_config.value` 为结构化配置（主存储）
- `config` 表中的 `scan_input_dirs` / `scan_cron` / `source_dir` / `target_dir` 为兼容镜像字段

## 配置写入规则

1. `PUT /api/config` 先读取现有配置并做字段级 merge（补丁语义）
2. 路径字段执行规范化与绝对路径校验（非空时）
3. 在单事务内同时写入 `app_config` 和 legacy `config` KV

## 路径与平台规则

- Linux（Docker/NAS）默认目录：
  - `SOURCE_DIR=/data/source`
  - `TARGET_DIR=/data/target`
  - `CONFIG_DIR=/data/config`
  - `DELETE_STAGING_DIR=/data/delete_staging`
- Win / macOS 默认目录：
  - 使用 `${HOME}/.classifier/*`
- 目录浏览接口 `GET /api/fs/dirs`：
  - 空 `path` 时自动落到平台根目录
  - 使用 API 返回的 `parent` 驱动前端“上级目录”导航

## 扫描目录来源优先级（手动触发扫描）

1. 已启用 `scan` 计划任务的 `source_dirs`（合并去重）
2. 配置项 `scan_input_dirs`
3. 配置项 `source_dir`
4. 环境变量 `SOURCE_DIR`（注入为默认回退）

## 迁移与兼容

- 启动阶段 `EnsureAppConfig`：
  - 若 `app_config` 不存在，则从 legacy `config` 映射生成
- `GetAppConfig`：
  - `app_config` 不存在时回退读取 legacy `config`
