## 端口类型速查

| 类型标识 | Go 常量 | 说明 |
|---------|---------|------|
| `PATH` | `PortTypePath` | 文件系统路径字符串 |
| `STRING` | `PortTypeString` | 普通字符串 |
| `STRING_LIST` | `PortTypeStringList` | 字符串列表 |
| `BOOLEAN` | `PortTypeBoolean` | 布尔值 |
| `JSON` | `PortTypeJSON` | 任意 JSON 对象 |
| `FOLDER_TREE_LIST` | `PortTypeFolderTreeList` | `[]FolderTree`，顶层目录树列表 |
| `CLASSIFICATION_SIGNAL_LIST` | `PortTypeClassificationSignalList` | `[]ClassificationSignal`，分类信号列表 |
| `CLASSIFIED_ENTRY_LIST` | `PortTypeClassifiedEntryList` | `[]ClassifiedEntry`，已分类条目列表 |
| `PROCESSING_ITEM_LIST` | `PortTypeProcessingItemList` | `[]ProcessingItem`，待处理项列表 |
| `MOVE_RESULT_LIST` | `PortTypeMoveResultList` | `[]MoveResult`，移动结果列表 |

图验证器仅在源端口和目标端口**均设置了类型**时才做兼容性检查，两端类型必须相同。

### 2. 目录树扫描器 · `folder-tree-scanner`

|------|---|
| **类型** | `folder-tree-scanner` |
| **标签** | 目录树扫描器 |
| **描述** | 递归扫描源目录，输出顶层子目录 FolderTree 列表 |

**输入端口**

| 端口名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `source_dir` | `PATH` | ✓ | 扫描根目录 |

**输出端口**

| 端口名 | 类型 | 说明 |
|-------|------|------|
| `tree` | `FOLDER_TREE_LIST` | 顶层子目录 FolderTree 列表 |

**配置项**

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `source_dir` | string | — | 静态指定扫描根目录（也可由输入端口动态传入，端口优先） |
| `max_depth` | int | `5` | 递归最大深度 |
| `exclude_patterns` | []string | 内置默认值 | 目录名排除模式（含隐藏目录、临时目录等） |
| `min_file_count` | int | `0` | 最少包含文件数，低于此值的目录被过滤 |

### 3. 关键词分类器 · `name-keyword-classifier`

| 属性 | 值 |
|------|---|
| **类型** | `name-keyword-classifier` |
| **标签** | Name Keyword Classifier |
| **描述** | Classify folder trees by configured name keywords |

**输入端口**

| 端口名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `trees` | `FOLDER_TREE_LIST` | — | 目录树列表 |
| `folder` | `FOLDER_TREE_LIST` | — | 目录树（兼容旧版，与 `trees` 取其一） |

**输出端口**

| 端口名 | 类型 | 说明 |
|-------|------|------|
| `signal` | `CLASSIFICATION_SIGNAL_LIST` | 命中关键词的分类信号列表 |
| `pass` | `FOLDER_TREE_LIST` | 未命中的目录树，透传给下一个分类器 |

**配置项**

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `rules` | []object | 内置规则 | 关键词规则列表，见下方 Rule 结构 |

**Rule 结构**

```json
{
  "keyword": "漫画",
  "category": "manga",
  "confidence": 0.9,
  "priority": 10,
  "match_type": "folder_name_contains"
}
```

`match_type` 当前仅支持 `folder_name_contains`（不区分大小写子串匹配）。未配置 `rules` 时使用内置默认关键词表。

### 4. 文件树分类器 · `file-tree-classifier`

| 属性 | 值 |
|------|---|
| **类型** | `file-tree-classifier` |
| **标签** | 文件树分类器 |
| **描述** | 根据 FolderTree 文件结构进行规则分类 |

**输入端口**

| 端口名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `trees` | `FOLDER_TREE_LIST` | — | 输入目录树列表 |
| `folder` | `FOLDER_TREE_LIST` | — | 目录树（兼容旧版） |

**输出端口**

| 端口名 | 类型 | 说明 |
|-------|------|------|
| `signal` | `CLASSIFICATION_SIGNAL_LIST` | 分类信号列表 |
| `folder` | `FOLDER_TREE_LIST` | 未命中透传目录树列表 |

**配置项**

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `rules` | []object | 内置规则 | 分类规则列表，见下方 Rule 结构 |

**Rule 结构**

```json
{
  "condition": "has_video_file",
  "category": "video",
  "confidence": 0.85
}
```

`condition` 支持的值：`has_video_file`、`has_image_file`、`mostly_images`、`mostly_videos` 等（由执行器内置匹配逻辑决定）。

### 7. 子树聚合器 · `subtree-aggregator`

| 属性 | 值 |
|------|---|
| **类型** | `subtree-aggregator` |
| **标签** | Subtree Aggregator |
| **描述** | Aggregate classification signals and persist folder category |

**输入端口**

| 端口名 | 类型 | 必填 | Lazy | 说明 |
|-------|------|------|------|------|
| `trees` | `FOLDER_TREE_LIST` | — | — | 目录树列表 |
| `node` | `FOLDER_TREE_LIST` | — | — | 目录树（兼容旧版） |
| `signal_kw` | `CLASSIFICATION_SIGNAL_LIST` | — | ✓ | 关键词分类器信号 |
| `signal_ft` | `CLASSIFICATION_SIGNAL_LIST` | — | ✓ | 文件树分类器信号 |
| `signal_ext` | `CLASSIFICATION_SIGNAL_LIST` | — | ✓ | 扩展名分类器信号 |
| `signal_high` | `CLASSIFICATION_SIGNAL_LIST` | — | ✓ | 置信度检查高置信度信号 |
| `signal_manual` | `CLASSIFICATION_SIGNAL_LIST` | — | ✓ | 人工分类信号 |

**输出端口**

| 端口名 | 类型 | 说明 |
|-------|------|------|
| `entry` | `CLASSIFIED_ENTRY_LIST` | 已分类条目列表 |

> 无配置项。聚合所有传入的信号，按优先级选出最终类别，并将结果持久化到数据库（更新 `folders.category`）。`Lazy` 端口表示该端口允许上游节点执行状态为 pending，聚合器仍可在其他输入就绪时运行。

### 13. 重命名节点 · `rename-node`

| 属性 | 值 |
|------|---|
| **类型** | `rename-node` |
| **标签** | Rename Node |
| **描述** | Transform target names by template, regex, or conditional rules |

**输入端口**

| 端口名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `items` | `PROCESSING_ITEM_LIST` | ✓ | 待重命名的处理项列表 |

**输出端口**

| 端口名 | 类型 | 说明 |
|-------|------|------|
| `items` | `PROCESSING_ITEM_LIST` | 已重命名的处理项列表 |

**配置项**

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `strategy` | string | `"template"` | 策略：`template` \| `regex_extract` \| `conditional` |
| `template` | string | — | 模板字符串，支持 `{name}`、`{ext}`、`{date}` 等变量 |
| `regex` / `pattern` | string | — | 正则表达式（`regex_extract` 模式） |
| `skip_if_same` | bool | `false` | 重命名结果与原名相同时跳过 |
| `rules` | []object | — | 条件规则列表（`conditional` 模式），每条规则含 `condition`（或 `if`）和 `template` |

**Conditional Rule 结构**

```json
{
  "condition": "category == 'manga'",
  "template": "[漫画] {name}"
}
```
