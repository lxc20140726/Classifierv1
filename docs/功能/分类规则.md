## 分类决策逻辑

分类策略以文件夹内容为准，保留文件夹原本结构，具体阈值可由用户在工作流中自定义。

**基础分类逻辑：**

```
扫描文件夹内所有文件
        |
        v
是否包含 .cbz/.cbr 文件？
或文件夹名含 漫画/comic/manga？
  YES -> Category = manga
  NO
        |
        v
统计 imageCount / videoCount
totalMedia = imageCount + videoCount
        |
        v
imageRatio = imageCount / totalMedia
videoRatio = videoCount / totalMedia
        |
        v
imageRatio >= 0.85  -> photo
videoRatio >= 0.85  -> video
两者均 >= 0.15      -> mixed
totalMedia == 0     -> unknown
```

**阈值说明：**

| 分类 | 条件 | 说明 |
|------|------|------|
| manga | 含 .cbz/.cbr 或名称关键词 | 优先判断 |
| photo | imageRatio >= 0.85 | 85% 以上是图片 |
| video | videoRatio >= 0.85 | 85% 以上是视频 |
| mixed | 两者均 >= 0.15 | 图片视频混合 |
| unknown | totalMedia == 0 | 无可识别媒体文件 |

## Magic Bytes 检测

仅对扩展名缺失或异常的文件执行 magic bytes 检测，避免不必要的 IO：

```go
var magicBytes = []struct {
    mime   string
    offset int
    magic  []byte
}{
    {"image/jpeg",       0, []byte{0xFF, 0xD8, 0xFF}},
    {"image/png",        0, []byte{0x89, 0x50, 0x4E, 0x47}},
    {"image/gif",        0, []byte{0x47, 0x49, 0x46, 0x38}},
    {"image/webp",       8, []byte{0x57, 0x45, 0x42, 0x50}},
    {"video/mp4",        4, []byte{0x66, 0x74, 0x79, 0x70}},
    {"video/x-matroska", 0, []byte{0x1A, 0x45, 0xDF, 0xA3}},
}
```

## 自定义分类规则（Phase 3）

用户可在设置页面添加自定义扩展名映射，扩展默认分类规则：

```json
{
  "custom_image_exts": [".jxl", ".qoi"],
  "custom_video_exts": [".iso"],
  "custom_keywords": {
    "manga": ["漫画", "comic", "manga", "scan"]
  }
}
```
