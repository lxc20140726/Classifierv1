# 可回退操作（Undo / Snapshot）

> 更新说明：若查看最新的节点级快照与断点续传设计，请优先阅读 [快照系统（版本3）](../功能/快照系统（版本3）.md)。

> 版本：v2.0 | 日期：2026-03-19

## 设计原则

- **操作前快照**：所有破坏性操作（重命名、移动）执行前，记录原始状态
- **快照粒度**：以单个文件夹为单位，记录操作前的文件路径列表
- **回退范围**：支持回退到任意历史操作节点
- **不可回退操作**：压缩（ZIP 生成是追加操作，原文件不变，无需回退）

## 数据模型

```go
type SnapshotStatus string
const (
    SnapshotPending   SnapshotStatus = "pending"
    SnapshotCommitted SnapshotStatus = "committed"
    SnapshotReverted  SnapshotStatus = "reverted"
)

type Snapshot struct {
    ID            string          `json:"id"`
    JobID         string          `json:"job_id"`
    FolderID      string          `json:"folder_id"`
    OperationType string          `json:"operation_type"` // rename|move
    Before        json.RawMessage `json:"before"`
    After         json.RawMessage `json:"after"`
    Status        SnapshotStatus  `json:"status"`
    CreatedAt     time.Time       `json:"created_at"`
}

type FileRecord struct {
    OriginalPath string `json:"original_path"`
    CurrentPath  string `json:"current_path"`
}
```

## 回退流程

```
用户触发 Revert(snapshot_id)
  |
  +-- 查询 snapshot.after（当前路径列表）
  |
  +-- 对每个文件：os.Rename(current_path, original_path)
  |
  +-- 全部成功
  |     +-- 更新 snapshot.status = "reverted"
  |     +-- 更新 folder 记录恢复原始状态
  |     +-- 写入 audit_log（action=revert, result=success）
  |
  +-- 任意失败
        +-- 回滚已重命名的文件
        +-- 写入 audit_log（action=revert, result=failure）
        +-- 返回错误
```

## API

```
GET    /api/snapshots?folder_id=&job_id=   # 查询快照列表
GET    /api/snapshots/:id                  # 快照详情（before/after diff）
POST   /api/snapshots/:id/revert           # 执行回退
```

## 前端设计

- **JobsPage** 每个已完成任务旁显示「撤销」按钮（仅 rename/move 类型）
- 点击撤销 → 弹出确认 Dialog，展示 before/after 文件路径对比表格
- 确认后调用 `/api/snapshots/:id/revert`，通过 SSE 实时显示回退进度
- 已撤销的任务显示「已撤销」badge，不可重复撤销
- 快照自动过期清理（Phase 3）：保留最近 30 天
